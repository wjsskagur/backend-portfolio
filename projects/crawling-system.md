# 정산 데이터 크롤링 안정화

> 기술: Python(Selenium), PHP

## 📌 문제
외부 레저·숙박 플랫폼에서 판매·취소되는 데이터를 크롤링으로 수집하는데,
기존 크롤링이 메모리 누수와 불필요한 프로세스로 인해 오류가 잦았다.
특히 **취소 데이터 누락**이 발생하면 정산이 어긋나 직접적인 매출 손실로 이어졌다.

## ⚙️ 해결
- **크롤링 안정화**: 기존 크롤링의 메모리 누수·불필요 프로세스를 리팩토링
  - 토큰 구조 분석으로 처리 가능한 채널은 직접 구현
  - 네이버 등 동적 페이지는 Selenium(Python)으로 대응
- **연동 구간 추적성 확보**: 타 업체 API 연동 구간에 전송 로그·네트워크 로그를 기록하여
  데이터가 어느 지점에서 누락되는지 추적 가능하게 구성

## 📈 결과
- **크롤링 오류 0건** 달성
- 취소·판매 데이터 유실 차단
- 개선 전 최대 약 **천만 원 규모로 발생하던 매출 손실이 개선 후 재발하지 않음**

## 💡 비고
단순히 크롤링을 고치는 데 그치지 않고, 연동 구간에 로그를 남겨
"문제가 생기면 어디서 생겼는지 알 수 있는" 추적 가능한 구조로 만든 것이 핵심이다.

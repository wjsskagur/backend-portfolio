# OMS PHP → Java 마이그레이션 및 사이트 리뉴얼

> 기간: 2024.08 ~ 2025.03
> 기술: Java, Spring Boot

## 📌 개요
파트너사 운영 시스템(OMS, Operation Management System)이 PHP와 Java가 혼용된
상태로 운영되고 있었다. 유지보수성과 일관성을 위해 **Java로 통합 마이그레이션**하고
사이트를 리뉴얼하는 대형 프로젝트를 진행했다.

## 🧩 역할
- PHP+Java 혼용 시스템의 Java 통합 마이그레이션 주도
- 사이트 리뉴얼 개발
- 프론트엔드 개발자·기획자와 3인 협업, **Swagger 기반 API 스펙 소통**

## ⚙️ 함께 진행한 작업

### 1. 크롤링 안정화
- 레저·숙박 채널 크롤링의 메모리 누수·불필요 프로세스를 리팩토링하여 오류 0건 달성
- 토큰 구조 분석으로 가능한 것은 직접, 네이버 등은 Selenium(Python)으로 대응

### 2. 연동 데이터 추적성 확보
- 타 업체 API 연동 구간에 전송 로그·네트워크 로그를 기록
- 데이터 누락 지점을 추적 가능하게 만들어 매출 손실 방지

## 📈 결과
- PHP+Java 혼용 → Java 통합으로 유지보수성 및 구조 일관성 확보
- 크롤링 오류 0건, 연동 데이터 유실 방지

## 💡 배운 점
혼용 스택을 한 번에 갈아엎기보다, 기능 단위로 점진적으로 옮기며 운영을 유지하는 것이
실무에서 안전하다는 것을 체감했다. Swagger로 프론트와 API 계약을 명확히 한 것이
협업 속도를 크게 높였다.

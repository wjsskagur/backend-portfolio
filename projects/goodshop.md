# 골프존 굿샵 (삼성물산 복지몰)

> 기간: 2022.06 ~ 2023.03 (런칭 후 약 6개월 운영 참여)
> 기술: Java, Spring Boot, JPA, MySQL

## 📌 개요
골프존 연동 쇼핑몰 구축 프로젝트. 웹비즈 재직 중 가장 큰 규모의 Java 프로젝트로,
**운영 환경에 JPA를 처음 본격 도입**했다. 팀 리딩을 맡아 설계부터 런칭, 운영까지 수행했고
실제 삼성물산 직원 복지몰로 런칭되어 운영 중이다.

## 🧩 역할
- 백엔드 아키텍처 및 프로젝트(언어/패키지) 구조 설계 주도
- DB 구조 설계
- 상품·회원·결제 도메인 개발
- 팀 리딩 및 일정 관리, 프론트엔드(Vue.js)팀과 API 협업

## 📌 문제 & 해결: JPA 페이징 데이터 중복

운영 환경에 JPA를 처음 도입하면서, **관리자–파트너사(`@OneToMany`) 관계를
페이징 조회할 때 부모(관리자) 데이터가 자식 수만큼 중복 증식되는 문제**를 겪었다.
컬렉션을 fetch join 하면 카테시안 곱이 발생해, 관리자 1명이 파트너사 수만큼
복제되어 조회되는 현상이었다.

**왜 발생했나**: 컬렉션 조인 + 페이징을 함께 사용하면, JPA가 정상적인 DB 페이징을
수행하지 못하거나 결과 행이 중복된다. (컬렉션 join fetch와 페이징은 함께 쓰기 어렵다.)

**해결**: 부모(관리자)와 자식(파트너사)을 한 번에 join fetch + 페이징하던 쿼리를
**둘로 분리**했다. 먼저 관리자만 페이징 조회하고, 조회된 관리자들의 파트너사를
별도 쿼리로 가져와 매핑하는 방식으로 바꿔 중복 증식과 N+1을 함께 해소하고
정상 페이징을 구현했다.

# AWS RDS 무중단 이관

> 기간: 2024.11 ~ 2025.02
> 기술: AWS DMS, HikariCP, CloudWatch

## 📌 문제
온프레미스 DB 서버에서 장애가 반복 발생하여 안정적인 운영이 어려웠고,
클라우드 기반으로의 전환이 필요했다.

## ⚙️ 해결
- **사전작업**: DB 커넥션 점검 중 잘못된 HikariCP 풀 설정을 교정하고, 미사용 프로젝트를 정리
- **이관**: AWS DMS를 활용해 무중단 이관 전략을 설계·실행
- **후속**: 이관 후 직접 구축하지 않은 영역의 쿼리 이슈까지 파악·수정하여 안정화
- 장애를 일간/주간/월간 보고서로 직접 관리하며 안정성 추적

## 📈 결과
- **CloudWatch 기준 DB CPU 사용률 35% → 18% (약 50% 개선)**
- 무중단 이관 완료로 서비스 중단 없이 전환
- 장애 발생 빈도 감소 및 운영 안정성 확보

## 💡 비고
DB CPU 수치는 AWS CloudWatch 지표 기준이며, 장애 추이는 직접 작성한
일간·주간·월간 장애 보고서를 근거로 관리했다.
